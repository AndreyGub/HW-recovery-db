### Ответ задание №1: 
# 1.1. Восстановление данных в полном объёме за предыдущий день
В нашем случае хорошо будет использование полного резервного копирования каждый день с дифференциальным или инкрементным копированием

# 1.2. Восстановление за час до предполагаемой поломки
В этом случае нужны более частые резервные копии, как вариант можно использовать  "журналы транзакций (Transaction Log Backups) + полное копирование" То есть это полное, раз в сутки, а каждые 15–60 минут — копии журналов транзакций. При восстановлении можно откатить базу на любой момент времени в пределах последнего часа до поломки. даёт возможность восстановить базу с точностью до минуты 
# 1.3. Возможен ли кейс, когда при поломке базы происходило моментальное переключение на работающую или починенную базу данных.
Да, возможен — это режим высокой доступности (синхронная репликация, кластеризация, автоматический failover), но он не заменяет резервное копирование. Она защищает только от аппаратных сбоев и падения сервера

### Задание №2:
# 2.1 Автоматизация процесса резервного копирования
Да процесс резервного копирования PostgreSQL можно автоматизировать.
Например: Использование cron + shell-скрипта
Это распространённый и гибкий способ на Linux-системах .

Вот пример скрипта для автоматического резервного копирования (/usr/local/bin/backup_postgres.sh):

bash
#!/bin/bash

# Настройки
BACKUP_DIR="/var/backups/postgres"
DATE=$(date +"%Y%m%d_%H%M%S")
DB_NAME="mydb"
DB_USER="postgres"
BACKUP_FILE="$BACKUP_DIR/${DB_NAME}_backup_$DATE.dump"
LOG_FILE="$BACKUP_DIR/backup.log"

# Создание директории для бэкапов 
mkdir -p $BACKUP_DIR

# Выполяем резервное копирование
pg_dump -U $DB_USER -F c -b -v -f $BACKUP_FILE $DB_NAME 2>> $LOG_FILE

# Далее проверяем
if [ $? -eq 0 ]; then
    echo "$(date '+%Y-%m-%d %H:%M:%S') - Backup successful: $BACKUP_FILE" >> $LOG_FILE
else
    echo "$(date '+%Y-%m-%d %H:%M:%S') - Backup FAILED for $DB_NAME" >> $LOG_FILE
fi

# Удаляем старые бэкапы (старше 30 дней)
find $BACKUP_DIR -name "*.dump" -mtime +30 -delete
Настройка прав и добавление в cron:

# Сделать скрипт исполняемым
sudo chmod +x /usr/local/bin/backup_postgres.sh

# Открываем crontab для редактирования
crontab -e

# Добавить строку для ежедневного выполнения в 3:00 ночи
0 3 * * * /usr/local/bin/backup_postgres.sh
