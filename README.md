### Ответ задание №1: 
# 1.1. Восстановление данных в полном объёме за предыдущий день
В нашем случае хорошо будет использование полного резервного копирования каждый день с дифференциальным или инкрементным копированием

# 1.2. Восстановление за час до предполагаемой поломки
В этом случае нужны более частые резервные копии, как вариант можно использовать  "журналы транзакций (Transaction Log Backups) + полное копирование" То есть это полное, раз в сутки, а каждые 15–60 минут — копии журналов транзакций. При восстановлении можно откатить базу на любой момент времени в пределах последнего часа до поломки. даёт возможность восстановить базу с точностью до минуты 
# 1.3. Возможен ли кейс, когда при поломке базы происходило моментальное переключение на работающую или починенную базу данных.
Да, возможен — это режим высокой доступности (синхронная репликация, кластеризация, автоматический failover), но он не заменяет резервное копирование. Она защищает только от аппаратных сбоев и падения сервера
